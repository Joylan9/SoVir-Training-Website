### ⚡ Backend Setup

```bash
# 1️⃣  Navigate to backend
cd SprachWeg/Backend

# 2️⃣  Install dependencies
npm install

# 3️⃣  Create environment file (see Environment Variables below)
cp .env.example .env

# 4️⃣  Start development server (hot-reload)
npm run dev

# 5️⃣  Build for production
npm run build && npm start
```

### ⚡ Frontend Setup

```bash
# 1️⃣  Navigate to frontend
cd SprachWeg/Frontend/sprachweg

# 2️⃣  Install dependencies
npm install

# 3️⃣  Create environment file
cp .env.example .env

# 4️⃣  Start Vite dev server
npm run dev

# 5️⃣  Build for production & deploy to Hostinger
npm run build
# Upload the dist/ folder to your Hostinger hosting panel
```

<details>
<summary><h2>💡 Key Design Decisions (Click to Expand)</h2></summary>

| Decision | Rationale |
|---|---|
| **Separate Skill vs Language models** | Different data structures — skills have categories, pricing tiers; languages have multi-level progressions |
| **OTP-based email verification** | More secure than click-through links; prevents bot registrations |
| **Dual auth (Email + Google OAuth)** | Reduces friction for Google users while maintaining traditional auth |
| **Virtual `isProfileComplete`** | Backend computes completeness; frontend enforces modal — clean separation |
| **Hostinger Hosting** | Reliable hosting with global distribution, HTTPS, and professional support |
| **Multer for uploads** | Server-side storage for course images and learning materials |
| **class-validator DTOs** | Type-safe, decorator-based request validation |
| **Helmet middleware** | Automatic security headers with zero config |
| **TailwindCSS 4 + Framer Motion** | Utility-first CSS + production-grade animations |

</details>
